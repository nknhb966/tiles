POINT SIZE
5
1点を1辺 [値×1mm] の立方体として描画するサイズ。 小さいほど精細だが、点同士の隙間が目立ちやすい。 大きいほど隙間が埋まり遠景が見やすくなるが、塊感が強くなる。
POINT DENSITY
100% (描画する点の割合)
各タイルの点群のうち何%を描画するかを指定する間引き設定。 100%で全点、50%で半分、10%なら1/10だけ描画。 タイル自体はそのままなのでメモリ・ダウンロード量は変わらないが、 描画コスト(特にGPU負荷)が直線的に下がる。 VRでカクつく / ハイエンドPCでも重いデータで滑らかさが欲しい時に下げる。 LODと違いタイル境界の粗さが目立たず、均一に薄くなるのが特徴。
WORLD SCALE
1.0× (1.0=実寸 / VR用に拡大)
世界の大きさを変える倍率。1.0 が実寸(メートル単位そのまま)。 VRで「小さく感じる」場合は 1.5〜2.0倍に上げると体感が改善する。 視点(カメラ)位置を中心に拡縮するので、目の前のものが視野から外れない。
LOD ERROR TARGET
20 (低=高詳細/重い)
表示詳細度のしきい値(画面上の許容誤差ピクセル)。 低いほど高精細なタイルが読み込まれるが、ダウンロード量と描画負荷が増える。 VRでカクつく時は値を上げ、PCで綺麗に見たい時は下げる。 目安: VR=20前後 / ハイエンドPC=10前後 / 低速モバイル=40前後。
CACHE TILES
2000 tiles (上限)
メモリ上に保持しておくタイル数の上限。 多いほど一度訪れた場所に戻った時の再ダウンロードが減って快適になるが、 RAM消費が増える。VRで街を歩き回って戻るような体験なら大きめが有利。 CACHE MEMORY と合わせて、どちらかの上限に達したら古いタイルから破棄される。
CACHE MEMORY
1536 MB (上限)
タイルキャッシュに割り当てるメモリ上限(MB)。 ハイエンドPCで重いデータを扱う場合は 2GB〜3GB程度まで上げると体感が向上。 モバイル/Quest等では 512MB前後が安全圏(超えるとブラウザに強制終了されることがある)。
PREFETCH
idle
現在のデータセットの全タイルを先読みしてブラウザのキャッシュに乗せる機能。 VR突入前など、視点移動でカクつかせたくない場合に有効。 タイル数が多いと相応の通信量と時間がかかる(実行前に確認ダイアログが出る)。
